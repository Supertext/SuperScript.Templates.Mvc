What Does It Do?

This library is an add-on for SuperScript. It is strongly recommended to read the documentation for the core project before proceeding with this library. If you haven't yet read the core documentation then it is recommended that you start by reading about the core concepts.

SuperScript.Templates.Mvc offers a Razor view-based block for enclosing HTML templates entities to be processed by SuperScript.

SuperScript will retrieve the contents of this block and create an instance of TemplateDeclaration containing these contents.

When the Razor view is rendered as a webpage the contents of the block will be empty and the retrieved contents will be emitted through the referenced IEmitter. 

	public static class HtmlExtensions
	{
		static IDisposable TemplateContainer(this HtmlHelper helper)
		static IDisposable TemplateContainer(this HtmlHelper helper, string name)
		static IDisposable TemplateContainer(this HtmlHelper helper, string name, string emitterKey)
		static IDisposable TemplateContainer(this HtmlHelper helper, int insertAt)
		static IDisposable TemplateContainer(this HtmlHelper helper, string name, int insertAt)
		static IDisposable TemplateContainer(this HtmlHelper helper, string name, string emitterKey, int insertAt)
	}
Name Type Description
helper HtmlHelper

The context HtmlHelper object exposed on a Razor view.

emitterKey string

Indicates which instance of IEmitter the content should be added to.

Optional. If not specified then the contents will be emitted through the default instance of IEmitter.

Read more about the default emitter here.

insertAt int

Indicates the index in the collection at which the contents are to be inserted.

Optional. If not specified, like all declarations, this declaration will be appended to the Collection.

Note that a declarable may be added at the specified index in the Collection at the time when this declaration is being processed; this is notwithstanding any later declarations which may also be inserted at the same index.

name string

The client-side name assigned to this template.

Example

	@using SuperScript.Templates.Mvc.Containers
	…
	@using (Html.TemplateContainer("demoTemplate", "templates", 0))
	{
		
  • ${Name}
    • 
	}

    The above example creates a single instance of TemplateDeclaration with demoTemplate as the name of the client-side template.

    The following example shows two templates being declared inside the same container. 

    	@using SuperScript.Templates.Mvc.Containers
	…
	@using (Html.TemplateContainer("templates", 0))
	{
		<script type="text/html" id="testTemplate">
			<li>${Name}</li>
		</script>
		
		<script type="text/html" id="testTemplate2">
			<li>Name: ${Name}</li>
		</script>
	}

    An inner <script> tag is optional when only one template is being declared, but in the case where multiple templates are being declared then these templates must be segregated within <script> tags. The client-side name for each template is therefore declared on each <script> tag.

    The client-side name may be declared with either an id or a name attribute. 

      <script type="text/html" id="testTemplate">
  
  // or
  
  <script type="text/html" name="testTemplate">

    The examples above will move each HTML template into a TemplateDeclaration declaration. These declarations will be emitted through the an IEmitter with the Key "templates". (Note that this requires that an IEmitter with the Key "templates" is configured within the project. This is not provided out of the box by SuperScript.)

    Because of the 0 supplied in the TemplateDeclaration declaration, the declaration created in the above example will be added as the first item in the Collection. Note that this declarable will only be assured as the first item in the Collection at the time it is being processed; this is notwithstanding any later declarations which may also be inserted at the same index.
